## 🔍 What is Bio-SentinelX?

Bio-SentinelX is an **AI-based preventive health intelligence system** that acts as an early-warning mechanism for environmental health risks. Instead of just displaying weather data, it interprets how atmospheric conditions — temperature, humidity, air quality, UV radiation, pollen, and more — directly impact human health outcomes.

The platform fuses **real-time environmental data**, **machine learning prediction**, and **large language models** to forecast disease risk, alert users to respiratory hazards, heat stress, vector-borne infections, and allergen exposure — *before symptoms appear*. By shifting healthcare from reactive treatment to **proactive prevention**, Bio-SentinelX empowers individuals, communities, and public health systems to act in time.

## ✨ Core Features

### 🌤️ Real-Time Environmental Intelligence
- Live weather data: temperature, feels-like, humidity, wind speed/direction, cloud cover, pressure, visibility, dew point
- Advanced atmospheric data: UV index (hourly + daily max), wet-bulb temperature, CAPE, boundary layer height, soil moisture, vapour pressure deficit
- Air quality tracking: PM2.5, PM10, CO, CO₂, NO₂, SO₂, O₃, AQI (US standard)
- Pollen counts: alder, birch, grass, mugwort, olive, ragweed
- Hourly & 7-day forecast with precipitation probability
- Official weather alerts (severe events, storms, heatwaves)

### 🤖 ML-Driven Disease Risk Prediction
- Connects to a **FastAPI ML backend** (hosted on Railway) trained on environmental-health datasets
- Classifies risk into: `LOW`, `MODERATE`, `HIGH`, `CRITICAL`, `UNCERTAIN`
- Identifies the **primary disease trigger** (respiratory, heat stress, infection, vector-borne, etc.)
- Returns **confidence scores**, top contributing features, and explainability factors
- Supports user feedback loop for continuous model improvement

### 🧠 AI Health Risk Assessment (MedGemma-First Routing)
- Deep health analysis generated by LLMs using real environmental context
- Supports **8 AI providers** with model selection:
  - **Hugging Face MedGemma** (MedGemma 4B IT, MedGemma 27B Text IT, Indian Meds adapter)
  - **Google Gemini** (Gemini 3 Flash, 2.5 Pro, 2.5 Flash, 2.0, 1.5 Pro/Flash)
  - **Groq** (Llama 3.3 70B, Llama 3.1 8B, Mixtral 8x7B, Gemma 2 9B)
  - **Cerebras** (llama-4-scout, llama-3.3-70b — with automatic 128-token block prompt caching)
  - **SiliconFlow** (DeepSeek V3, Qwen3, MiniMax, Kimi K2 and more)
  - **OpenRouter** (Claude 3.5, Gemini 2.5, DeepSeek R1, and 50+ models)
  - **Pollinations** (GPT-4o, o3 Mini, Qwen 2.5, Llama, Mistral, DeepSeek)
  - **Ollama** (on-device small models)
- Report/chat routing now prioritizes MedGemma when HF token is present, then falls back to Gemini and Groq as needed
- Reports rendered as interactive, collapsible sections with a table of contents
- Downloadable and printable health reports

### 💬 BioX Assistant (AI Chat + Deep Analysis)
- Context-aware health chatbot with **persistent session memory**
- Knows your current weather conditions, ML predictions, and personal health profile
- Generates personalised prompt suggestions based on your lifestyle and local conditions
- Supports multi-session conversation history
- User feedback on individual AI responses (helpful/not helpful + comments)
- Includes **Deep Analysis mode** with:
  - Knowledge Graph causal pathway reasoning
  - Auto research retrieval from indexed local documents
  - Optional MCP runtime tool execution with allowlist/timeout/retry guardrails
  - Live orchestration trace (`KG`, `RAG`, `MCP`, fallback path, final model/provider)

### 👤 Personal Health Profile
- Age, height, weight, gender
- **Live BMI calculator** with detailed health classification (7 categories from Severe Underweight to Obese Class III) and ideal weight range
- Lifestyle: sedentary / active / athlete / night shift / outdoor worker / remote work / frequent traveller
- Exercise level, smoking status, alcohol consumption
- Medications, food habits, known allergies
- Medical history (asthma, diabetes, hypertension, heart disease, COPD, migraine, arthritis, eczema, anxiety)
- Profile stored locally and incorporated into AI health analysis

### 🌊 Flood Prediction & River Risk Monitor
- Fetches 90-day historical + 60-day forecast river discharge data from **Open-Meteo Flood API**
- Computes a composite **flood risk score** based on discharge percentiles and seasonal context
- Detects trends (rising, stable, declining) with anomaly detection
- Seasonal awareness (monsoon/wet/dry season calibration per hemisphere)
- AI-generated flood situation analysis using LLM
- Interactive area/line charts for visualising historical discharge and forecasts

### 📅 Historical Climate & Health Analysis
- Fetches up to 1 year of historical weather data from **Open-Meteo Archive API**
- Selectable variables: temperature, precipitation, wind speed, humidity, thermal stress (apparent temperature), vapour pressure deficit, solar radiation, air quality (AQI)
- Interactive charts: line, area, and bar charts with referencing
- AI-generated health correlation analysis for the selected date range
- Quick date range selectors: last week / month / 3 months / year

### 📚 Research Library (RAG - Retrieval Augmented Generation)
- Upload and index health/medical research documents
- Supports: plain text paste, `.txt` / `.md` / `.csv` file upload, PDF via **LlamaCloud**, and URL scraping via LlamaCloud
- Documents are chunked, embedded (Gemini embedding model or TF-IDF fallback), and stored in an **in-browser vector database**
- Semantic retrieval for relevant research during health analysis and Deep Analysis assistant mode
- Re-embed and refresh individual documents
- Accessible directly from Deep Analysis via the in-assistant Knowledge panel

### 🧭 Agentic Deep Analysis + MCP
- Agentic orchestration combines weather context, ML prediction, KG reasoning, and RAG evidence
- MCP execution is guarded by runtime policies configured in Settings:
  - tool allowlist
  - timeout budget
  - retry budget
  - enabled server selection
- Execution trace is surfaced in assistant UI for auditability and debugging

### 📊 Charts & Data Visualisation
- Recharts-powered: line charts, area charts, bar charts with reference lines
- Risk score gauges, probability breakdowns, feature importance rankings
- Historical trend visualisation with anomaly highlighting
- Daily forecast temperature curves with precipitation probability overlay

### 🌙 Theme & Display Customisation
- **Light / Dark / Auto (OS-sync)** app theme
- Weather card mode: light / partial-dark / full-dark (auto-switches by local time)
- Custom accent colour, surface colour, and text colour picker
- Weather theme lock/unlock

### � AI Smart Notifications
- **20+ rule-based health alerts** fire automatically from live sensor data (temperature, AQI, UV, humidity, dew point, CAPE, wind)
- Each alert carries `severity` (critical / warning / info), a plain-language `message`, and an actionable `healthTip`
- **Session-aware** — morning / afternoon / evening context for time-appropriate advice
- AI **rewrites** the generated alert title & body on the fly for specificity: includes the exact measured value and city name
- **Provider waterfall**: Pollinations (no key needed) → Groq → OpenRouter → SiliconFlow → hardcoded fallback — never silently fails
- **LRU cache** (60 slots, keyed by `severity|category|value`) prevents repeated API calls for the same alert condition

### �💾 Smart Memory & Report Storage
- Reports auto-saved with city, timestamp, risk score, and primary risk tag
- Report history viewer with collapsible section rendering
- Session-level and cross-session conversation memory
- Memory summaries persisted in localStorage

### ⚡ Quick Health Check
- One-tap health snapshot for the current location
- Instant feedback on dominant risk factors without full analysis

### 🇮🇳 Indian Syndromic Surveillance (Phase 1 — Data Acquisition)
- **Indic language support**: Processes informal clinical descriptions in Hindi, Hinglish, Bhojpuri, and 22 scheduled Indian languages
- **Data sources**: IndicLLMSuite/Sangraha (251B tokens), DISPLACE-M (Bhashini) field conversations, EpiClim district-wise outbreak data, IDSP weekly reports
- **Syndromic extraction**: Maps informal text (e.g., "बहुत तेज़ बुखार और ठंड लग रही है") to IDSR syndromes and ICD-10 codes
- **11 IDSP syndromes** monitored: Acute Watery Diarrhea, Acute Bloody Diarrhea, Acute Febrile Illness, Acute Respiratory Infection, Meningitis/Encephalitis, Measles, Acute Jaundice, Acute Flaccid Paralysis, Snake Bite, Dog Bite/Rabies, Unusual Fever Cluster
- **Zero-connectivity extraction**: Converts informal descriptions into structured WHO surveillance signals in 2-10 seconds without cloud dependency
- **Live preview**: Real-time syndrome/ICD-10 extraction as the health worker types
- **Enhanced local intel UX**: Duration, symptom trend, and exposure-factor capture with one-click handoff to Deep Analysis

### 📈 Outbreak Prediction (Phase 3 — Temporal Baseline)
- **Moving average baseline**: Calculates 4-week rolling mean (μ) and standard deviation (σ) per syndrome per district
- **Automated flagging**: When weekly cases N > μ + 2σ, the system flags a potential outbreak alert
- **Status levels**: Normal → Watch → Alert → Outbreak (escalating based on standard deviations above mean)
- **Climate integration**: Syndrome-specific risk multipliers from EpiClim variables (temperature, precipitation, humidity, LAI)
- **Seasonal context**: Monsoon-aware risk adjustment (e.g., waterborne disease risk ×1.8 during heavy rainfall)
- **District-level dashboard**: Per-syndrome status, trend direction (rising/stable/declining), and weekly history

### 🕸️ Environmental Health Knowledge Graph
- **In-browser graph**: 40+ nodes and 39+ evidence-backed edges mapping environmental factors → health outcomes
- **Causal chain traversal**: BFS algorithm finds all pathways from environmental conditions to health risks
- **Example chains**: High Humidity → Mold Growth → Asthma Trigger · Heavy Rainfall → Stagnant Water → Mosquito Breeding → Dengue
- **Live weather analysis**: Automatically identifies active environmental risk factors from current conditions and maps to health risks
- **Intervention suggestions**: Recommends evidence-based preventive actions (e.g., N95 masks for PM2.5, mosquito nets for vector-borne risk)
- **Explains the "Why"**: Provides causal reasoning for health reports with strength scores and scientific evidence

### 🔐 Privacy-First Architecture
- **On-device processing**: 90% of extraction and analysis runs locally on the device
- **Structured-only sync**: Only anonymized ICD-10 codes and syndrome categories are ever uploaded to cloud
- **PII redaction engine**: Automatically strips names, Aadhaar numbers, phone numbers, emails, IP addresses, and addresses
- **Consent management**: Granular controls for syndrome sync, cloud AI, analytics, and research data sharing
- **Privacy audit log**: Tracks every operation — where it ran (on-device vs cloud), what was anonymized, what was sent
- **Anonymization tester**: Interactive tool to verify PII stripping before any data leaves the device

### 🧬 Fine-Tuning Strategy (Phase 2 — MedGemma)
- **Base model**: MedGemma 1.5 4B optimized for medical reasoning and instruction-following
- **Methodology**: QLoRA (4-bit quantization) for fine-tuning on consumer-grade hardware via PEFT
- **Indian Meds adapter**: Builds on existing `medgemma-1.5-4b-it-sft-lora-indian-meds` baseline from Hugging Face
- **Document ingestion**: Training dataset creation from medical data, research papers, and clinical guidelines
- **Domain detection**: Automatic classification into respiratory, cardiovascular, heat stress, infectious, neurological, dermatological, and environmental domains
- **Ollama integration**: Fine-tuned models created via Ollama Modelfile system for local deployment

### 🔬 Multimodal Enhancements (Phase 4)
- **MedSigLIP integration**: Vision component from Google Health AI foundations for visual symptom analysis
- **Rural health imaging**: Handles rashes, water contamination photos, and hygiene assessment specific to Indian rural environments
- **Builds on existing YOLO/ResNet**: Extends current image analysis pipeline with medical-specific vision models
- **HeAR model support**: Google Health Acoustic Representations model (trained on 25,000 cough recordings) for respiratory classification

## ⚙️ AI Engine — Under the Hood

### 🗂️ Context Manager
> Governs every byte sent to the AI — no token is wasted.

- **Model Registry** — per-provider/model profile with context window, input budget, output limit, and temperature for reports vs. chat
- **Compact Weather Builder** — strips all `null`/`N/A` fields before building the prompt; only real sensor values go to the AI
- **Budget Allocation** — dynamically splits the token budget by section (weather 35%, dataset 30%, lifestyle 10%) per call type
- **Token Tracker** — every API call logged to `localStorage`; surfaces 24h stats in the Token Budget Panel (calls, tokens, saved, cached)

---

### 🗃️ Prompt Cache Service
> Two-layer caching to cut latency and redundant processing.

- **Client-side LRU** — system instruction strings memoized per call-type + city for 30 min; avoids rebuilding 2K–8K char strings on every call
- **Server-side accounting** — reads `usage.prompt_tokens_details.cached_tokens` from every API response to count provider-cached tokens (Cerebras, Groq, OpenAI-compatible)
- **Static content first** — system instructions and domain knowledge placed at prompt start to maximise prefix cache hit rate across all providers
- Stats (client hits, server hits, tokens saved) visible in the Token Budget Panel

---

### 🧠 Persistent Memory Service
> The app remembers across sessions — no login, no backend.

- **Report Chunking** — each health report parsed into named sections (`### Heat Stress`, `### AQI Analysis`, etc.) stored as individual TF-IDF-indexed chunks
- **Cross-Session Memory** — `MemorySummary` tracks last 5 cities, up to 15 key insight bullets, and up to 10 health concerns inferred from chat (e.g. asthma → "Respiratory sensitivity")
- **Session Summaries** — auto-generated 2-3 sentence summary every 4 messages; injected on next session start
- **Storage Safeguard** — LRU pruning when `localStorage` quota is hit; falls back gracefully without data loss

---

### 🔍 Vector Database (In-Browser RAG)
> Full retrieval-augmented generation with zero external vector DB.

- **Chunking** — research text split into ~650-char segments with 120-char overlap; snaps to sentence boundaries
- **Dual Embedding** — Gemini `text-embedding-004` for dense vectors; TF-IDF bag-of-words as a zero-API-key fallback
- **Semantic Search** — cosine similarity retrieval; top-k relevant chunks injected into the AI prompt at analysis time
- **Quota-safe Storage** — if `localStorage` is full, dense embeddings silently stripped; TF-IDF retained

---

### 🚨 Alert Engine
> Rule-based health alerts evaluated against live sensor data.

- **20+ rules** covering temperature, humidity, AQI, UV index, dew point, CAPE (storm potential), and wind
- Each alert returns `severity` (critical / warning / info), a plain-language `message`, and a concrete `healthTip`
- **Session-aware** — morning / afternoon / evening context for time-appropriate advice
- Alerts rendered in the UI with severity colour-coding and per-category icons

---

### 🔔 AI Notification Service
> Alert titles and messages rewritten live by AI for specificity and urgency.

- **Provider waterfall** — Pollinations (no key) → Groq → OpenRouter → SiliconFlow → hardcoded fallback
- **In-memory LRU cache** (60 entries keyed by `severity|category|value`) prevents redundant API calls for repeated alerts
- Prompt instructs the LLM to include the exact measured value and city; max 9-word punchy headline

## 📦 Getting Started

### Prerequisites
- Node.js 18+
- At least one AI provider API key (Gemini recommended — free tier available)

```bash
git clone https://github.com/gaur-avvv/Bio-SentinelX.git
cd Bio-SentinelX
npm install
npm run dev
```

### MedGemma (Hugging Face)

Use MedGemma for multimodal clinical reasoning workflows.

```bash
hf auth login
```

Pipeline usage:

```python
from transformers import pipeline

pipe = pipeline("image-text-to-text", model="google/medgemma-4b-it")
messages = [
  {
    "role": "user",
    "content": [
      {"type": "image", "url": "https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/p-blog/candy.JPG"},
      {"type": "text", "text": "What animal is on the candy?"}
    ]
  },
]
print(pipe(text=messages))
```

Direct model loading:

```python
from transformers import AutoProcessor, AutoModelForImageTextToText

processor = AutoProcessor.from_pretrained("google/medgemma-4b-it")
model = AutoModelForImageTextToText.from_pretrained("google/medgemma-4b-it")
messages = [
  {
    "role": "user",
    "content": [
      {"type": "image", "url": "https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/p-blog/candy.JPG"},
      {"type": "text", "text": "What animal is on the candy?"}
    ]
  },
]
inputs = processor.apply_chat_template(
  messages,
  add_generation_prompt=True,
  tokenize=True,
  return_dict=True,
  return_tensors="pt",
).to(model.device)

outputs = model.generate(**inputs, max_new_tokens=40)
print(processor.decode(outputs[0][inputs["input_ids"].shape[-1]:]))
```
